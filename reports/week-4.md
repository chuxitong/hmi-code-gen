# Неделя 4. Локальный HTTP-сервис вокруг модели

Автор: Чжан Сычэн.

Идея четвёртой недели была простая: плагин в Figma не должен сам запускать модель. Это и технически больно (в песочнице Figma никакого torch не будет), и архитектурно странно. Правильнее, чтобы плагин только формировал запрос, а модель крутилась в отдельном локальном процессе. Так и сделал.

Сервис живёт в `local-service/app.py` на FastAPI. Он слушает `http://localhost:8000` и поднимает четыре эндпоинта. `/generate` принимает base64-изображение фрейма и опционально переменные Figma и CSS-подсказки, возвращает HTML/CSS и сразу готовое превью PNG. `/refine` принимает референсное изображение, текущий код и возвращает улучшенный код с новым превью. `/edit` принимает текущий код и текстовую инструкцию. `/render` — служебный, просто превращает HTML в PNG через Playwright в headless-Chromium, нужен для превью и для экспериментов.

Модель подключается лениво: при первом запросе `app.py` пытается импортировать `model_wrapper.UI2CodeModel`, а если он по каким-то причинам недоступен, падает обратно на `rule_based_model.RuleBasedModel`. Это сознательное решение. Настоящая UI2Code^N на моей карточке грузится десятки секунд, поэтому для быстрой отладки пайплайна удобнее иметь лёгкую детерминированную заглушку. Обе модели дают один и тот же интерфейс `generate / refine / edit`, поэтому в остальном коде ничего не меняется.

Отдельно я прописал CORS, потому что плагин Figma при запросах видит себя как другой origin, и без CORS браузер просто блокирует fetch. Все ответы нормализованы в pydantic-модели `CodeResponse` / `RenderResponse`, чтобы плагин читал их без проверок на null.

Один тонкий момент всплыл с Playwright: браузеры нужно установить один раз командой `playwright install chromium`. Я добавил это в README и в `requirements.txt` как отдельный шаг.

В качестве демонстрации у меня в репозитории есть пример запроса к `/generate` с base64-PNG и пример ответа с полями `code`, `preview_base64`, `explanation`. Через Swagger UI на `http://localhost:8000/docs` всё это тоже доступно и проверяемо вручную.

По итогам недели плагин имеет стабильную серверную сторону. Плагин и сервис теперь реально разделены: плагин только про UX и данные из Figma, сервис про модель и рендер. На пятую неделю остаётся прикрутить рендер к превью плагина и собрать первую полную оболочку UI.
