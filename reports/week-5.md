# Неделя 5. Рендер кода в картинку и первая рабочая оболочка плагина

Автор: Чжан Сычэн.

Пятая неделя ушла на две параллельные задачи. Первая — сделать так, чтобы сгенерированный HTML превращался в картинку, которую можно показать рядом с референсом Figma. Вторая — собрать финальный UI плагина с тремя рабочими действиями, а не просто заготовкой кнопок.

Рендер получился через Playwright. В `local-service/renderer.py` есть одна функция `render_html_to_png`, которая открывает HTML во временном файле, запускает headless-Chromium с нужным viewport и снимает скриншот страницы. Эту же функцию использует эндпоинт `/render`, и через неё же сервис формирует `preview_base64` в ответах других эндпоинтов. Важный момент — viewport. Мы рисуем 1280×720 по умолчанию, это близко к формату промышленных экранов и совпадает с размерами мокапов из недели 2.

Плагин я компилирую стандартным `npx tsc` из `figma-plugin/`. На выходе получается `figma-plugin/dist/code.js`, который и грузится Figma. В `src/code.ts` обработчики сообщений делают всё, что нужно: проверяют, что выделен ровно один фрейм, экспортируют его через `exportAsync` в PNG, при включённых галках собирают локальные переменные и CSS-подсказки, и шлют всё это в UI. Дальше UI в `src/ui.html` уже самостоятельно делает fetch к локальному сервису, показывает полученный код в textarea, рендерит превью через `/render` и пишет лог действий в нижнюю область.

В финальном UI получилось именно то, чего хотелось: сверху кнопка Generate Code и кнопка Make It Closer to the Mockup, между ними две галки (переменные дизайна и CSS-подсказки), ниже блок Edit by Request с полем ввода инструкции и кнопкой Apply Edit, область с кодом и кнопкой Copy Code, область превью и область лога. Цветовая схема подобрана под тёмную тему Figma, чтобы не слепило.

Связь плагин↔сервис я проверил тремя способами. Сначала ping/pong на старте (плагин шлёт в UI сообщение, UI отвечает через postMessage). Потом реальный экспорт фрейма и вызов `/generate`: я открывал мокап `01-equipment-status`, жал Generate Code, получал HTML и превью в плагине. Потом пробовал `/refine` и `/edit`, чтобы убедиться, что полный тройственный сценарий работает.

Из трудностей: в manifest.json надо было явно разрешить `http://localhost:8000`, иначе Figma блокирует fetch. И Playwright всё-таки требует ручной команды установки Chromium один раз, это в README есть. Остальное прошло без боли.

По итогам у меня собранная оболочка плагина, рабочий рендер-модуль, и первый end-to-end прогон от выделения фрейма до готового HTML с превью. Для отчёта подготовлены иллюстрации панели и короткий GIF с тремя кадрами в `reports/screenshots/week05_plugin_panel.png` и `reports/screenshots/week05_pipeline_demo.gif`. Шестая неделя строится на этом и закрывает полный сценарий Generate + Refine по всем мокапам.
