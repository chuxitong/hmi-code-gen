# Неделя 1. Знакомство с Figma Plugin API и структура проекта

Автор: Чжан Сычэн.

На первой неделе я в основном разбирался с тем, как вообще устроен плагин Figma и какие данные оттуда можно вытащить. До этого момента я писал только обычные веб-приложения, поэтому разделение «код в песочнице + интерфейс в iframe + обмен через postMessage» пришлось привыкать. По итогу получилось поднять минимальную рабочую структуру репозитория и зафиксировать, что из Figma нам реально понадобится.

Репозиторий разложен так, чтобы каждая часть жила отдельно и чинилась отдельно. В `figma-plugin/` лежит сам плагин, в `local-service/` — будущий HTTP-сервис, в `mockups/` — макеты, в `baseline-tests/` — место под эксперименты, в `reports/` — эти самые отчёты, в `docs/` — техническая документация. Такая структура упрощает работу руководителю: достаточно зайти в нужную папку, чтобы увидеть артефакты конкретного этапа.

Основной вопрос недели был, какие данные из Figma обязательны, а какие опциональны. Я решил сделать обязательным экспорт выбранного фрейма в PNG через `exportAsync`, метаданные фрейма (имя, размер), и двусторонний обмен сообщениями плагин-UI через `figma.ui.postMessage` и `parent.postMessage`. Как опциональные каналы добавил переменные дизайна (`figma.variables`) и CSS-подсказки (`getCSSAsync`). Опциональные — потому что они нужны не всегда, и модель должна уметь работать даже без них.

В качестве proof-of-concept написал `figma-plugin/src/code.ts` и `figma-plugin/src/ui.html`. Песочница экспортирует выбранный фрейм, кодирует его в base64, собирает CSS и переменные (если пользователь включил соответствующие галки) и шлёт всё в UI. UI уже показывает заготовки кнопок Generate / Refine / Apply Edit, текстовое поле для правок и область превью. Связь песочница↔iframe проверена через ping/pong, работает.

Из проблем: `getCSSAsync` не во всех версиях Figma API присутствует, поэтому вокруг его вызова стоит try/catch с мягким fallback. Для полного теста плагина нужен Figma Desktop, но загруженный плагин проверен вручную.

Что доставил за неделю: README с описанием проекта, техническую записку по извлечению данных из Figma, манифест, конфигурацию сборки, и сам исходник плагина. На следующей неделе планирую собрать тестовый набор промышленных мокапов.
