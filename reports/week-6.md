# Неделя 6. Generate Code и первая версия Refine по всем мокапам

Автор: Чжан Сычэн.

Шестая неделя была про то, чтобы всё, собранное за предыдущие недели, реально заработало вместе от начала до конца. Звучит просто, но обычно именно на сшивке таких систем всплывает куча мелких проблем.

Сквозной сценарий Generate Code у меня в итоге выглядит так. Пользователь в Figma выделяет один фрейм и жмёт кнопку. Плагин проверяет, что выделение корректно (нельзя сразу несколько фреймов или голый layer-group), экспортирует выделенный фрейм через `exportAsync` с `constraint: { type: "SCALE", value: 2 }` — двукратное увеличение даёт более читаемую картинку для модели. Если пользователь включил галки, плагин дополнительно подтягивает переменные дизайна и CSS-подсказки по иерархии выделения. Потом всё это base64-изображение и опциональный контекст уходят в сервис через `POST /generate`. Сервис вызывает модель, получает HTML, прогоняет его через `/render` и возвращает обратно пару «код + превью PNG». В плагине код попадает в textarea, превью в блок картинки, и можно нажать Copy Code.

Для первой версии Make It Closer to the Mockup я ничего нового в архитектуру не добавлял. Там достаточно, что плагин помнит исходный PNG мокапа, берёт текущий код из textarea, запрашивает у сервиса `/render` актуальный рендер, и потом шлёт все три вещи на `/refine`. Модель в режиме polishing смотрит на пару «референс — рендер» и правит код. Обычно после одной-двух итераций видно, что отступы становятся ближе к оригиналу и акценты расставляются правильнее. На третьей итерации эффект уже незаметный, иногда даже хуже, поэтому в UI я не стал ставить автоматический цикл и оставил одну итерацию на клик.

Чтобы не гонять мокап-за-мокапом руками, я написал отдельный скрипт `baseline-tests/run_full_evaluation.py`. Он берёт все восемь мокапов из `mockups/png/`, для каждого прогоняет три триала генерации, потом две итерации уточнения и одну итерацию правки по запросу. По пути он считает тайминги, сохраняет HTML и PNG результатов в `baseline-tests/outputs/full-eval/<mockup>/` и в конце пишет сводку в `metrics.json`, `metrics.csv` и `summary.md`. Такой пакетный прогон удобен и мне (чтобы видеть динамику), и руководителю (чтобы проверить итог одним взглядом).

Реальные наблюдения после прогонов. Простые экраны — обзор оборудования, таблица аварий — модель собирает практически без замечаний, правки на refine уходят в улучшение отступов и усиление визуальной иерархии. Средние — тренды, операторская панель, обзор линии — требуют одной итерации refine, чтобы пропорции карточек выглядели адекватно. Сложные — синоптика резервуаров и экран рецептов — это там, где честно видно ограничения модели: отдельные SVG-элементы (силуэты танков, клапаны, схема трубопровода) модель либо упрощает до прямоугольников, либо просто теряет. Это вписывается в ожидания из недели 3.

По итогам у руководителя на проверку есть `run_full_evaluation.py` как артефакт, `metrics.json` и `summary.md` с таймингами, и по четыре фазы на восемь мокапов (HTML и PNG для каждой фазы). Плагин на этом этапе устойчиво выполняет полный цикл Generate → Refine (в refine на сервер уходят референсное изображение, рендер текущего кода и сам код). Для наглядности в отчёт положены два **разных** трёхпанельных коллажа (не дублируют кадры недели 5): `week06_m1_reference_generate_refine2.png` — **Production Line Overview** (`05-production-overview`), `week06_m4_reference_generate_refine2.png` — **Operator Panel** (`04-operator-panel`); в каждой панели подписи: референс, первичный generate, refine ×2.

На седьмой неделе я берусь за Edit by Request и сравнение уровней контекста.
